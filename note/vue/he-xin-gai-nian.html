    核心概念

    党洁 1/8/2022 Vuex

    Vuex使用单一状态树，每个应用只包含一个store实例

    # 一、State

    Vuex中的State和组件中的data类似，遵循相同的规则。

    # 1.在Vue组件的computed中获得state。

    computed:{
    count () {
      return  this.$store.state.count
    }
}

    # 2.mapState辅助函数

    当需要获取多个state的时候，我们可以使用 mapState 辅助函数帮助我们生成计算属性。

    import {mapState} from 'vuex'

computed: {
    localComputed () { /* ... */ },

    // 使用对象展开运算符将此对象混入到外部对象中
    ...mapState('user[命名空间]',['count','user'])

}

    # 二、getters

    Vuex 允许我们在 store 中定义getter,从state派生出的状态（可以认为是 store 的计算属性）。

    # 1.getters的参数和返回值

    • getters有两个参数，state和getters
    • getter 可返回一个函数，来实现给 getter 传参。
    const store = createStore({
  state: {
    todos: [
      { id: 1, text: '第一篇', done: true },
      { id: 2, text: '第二篇', done: false }
    ]
  },
  getters: {
    //有两个参数，state和getters
    doneTodos: (state) => {
      return state.todos.filter(todo => todo.done)
    },
    doneTodosCount (state, getters) {
    return getters.doneTodos.length
     },
    //可以返回一个函数，来实现给getter传参
    getTodoById: (state) => (id) => {
    return state.todos.find(todo => todo.id === id)
  }
  }
})

    我们可以很容易地在任何组件中使用它。在computed中获得getters

    computed:{
    getTodoById(){
    return this.$store.getters.getTodoById(2)
    }
}

    # 2.mapGetters辅助函数

    mapGetters 辅助函数仅仅是将 store 中的 getter 映射到局部计算属性：

    import { mapGetters } from 'vuex'

export default {
  computed: {
  // 使用对象展开运算符将 getter 混入 computed 对象中
    ...mapGetters(
        //'命名空间',
    [
      'doneTodosCount',
      'anotherGetter'
    ])
  }
}

    # 三、mutation

    更改store 中的状态的唯一方法是提交 mutation。mutation类似于事件，事件有两个参数，state和payload（载荷）。

    mutation必须是同步函数

    # 1.在组件中提交（commit）mutation

    // ...
mutations: {
    //函数有两个参数，state和payload
    increment (state, payload) {
    state.count += payload.amount
  }
}

    methods:{
    increment(){
        this.$store.commit('increment',{amount:1})
    }
}

    # 2.mapMutations辅助函数

    可使用 mapMutations 辅助函数将组件中的 methods 映射为 store.commit 调用。

    import { mapMutations } from 'vuex'

export default {
  // ...
  methods: {
    ...mapMutations([
      // `mapMutations` 支持载荷：
      'increment' // 将 `this.increment(amount)` 映射为 `this.$store.commit('increment', amount)`
    ]),
    ...mapMutations({
      add: 'increment' // 将 `this.add()` 映射为 `this.$store.commit('increment')`
    })
  }
}

    # 四、actions

    Action 函数接受一个与 store 实例具有相同方法和属性的 context 对象和payload。

    action类似mutation，不同点是：

    • action 提交的是 mutation，而不是直接变更状态。
    • action 可以包含任意异步操作。

    # 1.在组件中分发（dispatch）action

    const store = createStore({
  state: {
    count: 0
  },
  mutations: {
    increment (state) {
      state.count++
    }
  },
  actions: {
    incrementAsync ({ commit }) {
    setTimeout(() => {
      commit('increment')
    }, 1000)
  }
  }
})

    methods:{
    incrementAsync(){
        this.$store.dispatch('incrementAsync')
    }
}

    # 2、mapActions辅助函数

    使用 mapActions 辅助函数将组件的 methods 映射为 store.dispatch 调用

    import { mapActions } from 'vuex'

export default {
  // ...
  methods: {
    ...mapActions({
      add: 'increment' // 将 `this.add()` 映射为 `this.$store.dispatch('increment')`
    })
  }
}

    # 3、组合Action

    actions: {
  actionA ({ commit }) {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        commit('someMutation')
        resolve()
      }, 1000)
    })
  }
}

    # 五、modules

    Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter。

    const moduleA = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = createStore({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

    # 模块的局部状态

    • state是局部的，getters、mutations、actions仍然是全局命名空间的,通过添加 namespaced: true 的方式使所有选项局部空间化。
    • getters、mutations接受的第一个参数是模块局部state，actions中context.state是局部状态，context.rootState是根节点状态。
