    Vue组件

    党洁 1/1/2022 Vue

    组件是可复用的 Vue 实例,所以它与Vue实例接收相同的选项，除了el。

    组件只有一个根元素。

    每用一次组件，就会有它的一个实例被创建。

    组件中的data必须是函数，因此每个实例可以维护一份被返回对象的独立的拷贝，不会相互影响。

    # 全局注册和局部注册

    Vue.component 是全局注册的，它们在注册之后可以用在任何新创建的 Vue 根实例 (new Vue) 的模板中。

    在实例的components 选项中定义你想要使用的组件，是局部注册。

    注意全局注册的行为必须在根 Vue 实例 (通过 new Vue) 创建之前发生。

    # 组件的props

    父组件通过 Prop 向子组件传递数据。 子组件通过$emit触发事件，父组件通过v-on监听子组件事件，实现父子组件之间的沟通

    示例

    Vue.component('blog-post', {
  props: ['title'], //数组形式的props
  template: 
  '<div>   //组件只有一个根元素
    <h3>{{ title }}</h3>
    <button v-on:click="$emit('enlarge-title',0.1)"></button>
   </div>'
})

    prop 被注册之后，你在父组件可以像这样把数据作为一个自定义 attribute 传递进来

    <div :style="{ fontSize: postFontSize + 'em' }">
    <blog-post v-bind:title="parentTitle" :value="parentLikes" @enlarge-title="postFontSize+= $event"></blog-post>
</div>
  data:{
   postFontSize:5
  }

    # prop

    # prop类型

    prop: {
      //设置prop的类型
      type: [Object,String],  
      // 必填
      required:true,         
      // 对象或数组默认值必须从一个工厂函数获取,默认值
      default: function () {
        return { message: 'hello' }
      },
      //验证
      validator: function (value) {
        return ['success', 'warning', 'danger'].indexOf(value) !== -1
      }
    },

    # 非 Prop 的 Attribute

    一个非 prop 的属性 是指传向一个组件，但是该组件并没有相应 prop 定义的 attribute。这个属性会被添加至组件的根元素。

    如果你不希望组件的根元素继承 attribute，你可以在组件的选项中设置 inheritAttrs: false

    $attrs:接收除了props声明外的所有绑定属性（class、style除外）。并且通过 v-bind="$attrs"， 可以将属性继续向下传递，让 grandson.vue 也能访问到父组件的属性。

    图例 

    Vue.component('base-input', {
  inheritAttrs: false,
  props: ['label', 'value'],
  template: `
    <label>
      {{ label }}
      <input
        v-bind="$attrs"  //将非prop的属性添加至input元素，而非根元素上。
        v-bind:value="value"
        v-on:input="$emit('input', $event.target.value)" 
      >
    </label>
  `
})

    $listeners:接收除了带有.native事件修饰符的所有事件监听器,通过 v-on="$listeners"，将事件监听器继续向下传递，让 grandson.vue 访问到事件

    # prop的单向数据流

    prop的单向数据流：父级prop更新会向下流动到子组件中，但反之不行。防止从子组件意外变更父级组件的状态。所以不应该在一个子组件内部改变 prop

    1.prop 用来传递一个初始值；这个子组件接下来希望将其作为一个本地的 prop 数据来使用。

    props: ['initialCounter'],
data: function () {
  return {
    counter: this.initialCounter
  }
}

    2.prop 以一种原始的值传入且需要进行转换。

    props: ['size'],
computed: {
  normalizedSize: function () {
    return this.size.trim().toLowerCase()
  }
}

    # 动态组件&&异步组件

    在不同组件之间动态切换 可以通过 Vue 的 component元素加一个特殊的is属性来实现。

    在动态组件上使用keep-alive,可以将失活的组件缓存

    <keep-alive>
  <component v-bind:is="currentTabComponent"></component>
</keep-alive>

    currentTabComponent 可以包括

    • 已注册组件的名字
    • 一个组件的选项对象
    //异步组件
new Vue({
  // ...
  components: {
    'my-component': () => import('./my-async-component')
  }
})

    # 通过插槽分配内容

    父组件在引用子组件的时候，希望传递模板内容。slot就是一个槽，接收结束标签和开始标签中的所有内容

    # 1.默认插槽（后备内容）

    <!--父组件中引用子组件，并传递内容-->
<submit-button>
  Save
</submit-button>

    <!--Submit作为后备内容，将它放在 <slot> 标签内。若父组件不传递任何东西，则使用Submit；否则使用父子间提供内容。-->
<button type="submit">
  <slot>Submit</slot>
</button>

    # 2.具名插槽

    子组件中有多个插槽，通过给插槽指定名称方式实现一一对应。 父组件在 template 元素上使用 v-slot 指令，并以 v-slot 的参数的形式提供其名称：

    <base-layout>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</base-layout>

    子组件中slot元素有一个属性：name。 可以用来定义额外的插槽：

    <div class="container">
  <header>
    <slot name="header"></slot>
  </header>
  <main>
    <slot></slot>
  </main>
  <footer>
    <slot name="footer"></slot>
  </footer>
</div>

    # 作用域插槽

    父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。 把子组件作用域中的数据传到父组件的插槽中使用

    # 插槽prop

    子组件可以自身作用域的数据作为slot元素的一个属性绑定上去：

    <span>
  <slot v-bind:user="user">
    {{ user.lastName }}
  </slot>
</span>
...
data:{
  user:
}

    父组件两个属性合并成了一个v-slot : 插槽名称 = ' 传过来的值 '，只能用在template元素上，但如果是只有默认插槽时，v-slot可以用在组件上。

    <current-user v-slot:default="slotProps">
  {{slotProps.user.firstName}}
</current-user>

<!--v-slot简写#，必须在其有参数的时候才可用-->
<!--解构插槽prop-->
<current-user #default="{user:person}">
  {{person.firstName}}
</current-user>

<!--你甚至可以定义后备内容，用于插槽 prop 是 undefined 的情形：-->
<current-user v-slot="{ user = { firstName: 'Guest' } }">
  {{ user.firstName }}
</current-user>

    # 动态插槽名称

    <base-layout>
  <template v-slot:[dynamicSlotName]>
    ...
  </template>
</base-layout>
